搭建个人博客那些事

网上有一些平台提供了个人博客的业务，比如CSDN，简书，LOFTER……方便使用，有个账号就行了。但是这些平台提供的样式都是固定的，如果想拥有个人博客的话还是需要自己的动手搭建的。

###那么搭建个人博客一般有哪些方式？
目前搭建个人博客有常用的两种方式：WordPress 和 GitHub Pages。下面就来说说这两种搭建博客的方式。

我们采用比较的形式来说这两种方式：

  • 复杂程度

    • WordPress搭建简单，近乎傻瓜式。
    • GitHub Pages较为复杂，需要安装一些软件，并且在搭建的过程中可能会出现各种莫名的错误，需要随时Google解决。

  • 空间

    • WordPress需要自己购买虚拟主机，如果遇到空间商跑路的话，比较O疼。
    • GitHub Pages空间免费，使用GitHub托管，稳定安全。

  • 域名

    • WordPress需要自己购买域名
    • GitHub Pages提供给免费的域名，但是也能够绑定顶级域名。赞！

  • 编写博客

    • WordPress需要在后台编辑文章，需要联网。
    • GitHub Pages支持本地编写、本地预览，不需要联网，可以使用git命令同步来管理文章、版本控制等等。支持MarkDown语法。

  • 主题、工具插件

    • WordPress的主题和插件是非常丰富的，不过质量良莠不齐，需要自己选择。就好比安卓用户刷机的时候，各种版本应有尽有，但是需要自己安装之后感受，不行就换下一个……
    • GitHub Pages的主题和插件相对WordPress来说少一些，但是越来越丰富，基本也够用。

  • 访问速度

    • WordPress的访问速度与你购买的虚拟空间有关。
    • GitHub Pages的访问速度说快不快，说慢不慢，但是稳定些。

  • 迁移成本

    • WordPress的迁移需要将网站目录迁移，数据库备份后再导入即可。
    • GitHub Pages的话自己有一套ruby和nginx的环境的话也是可以很方便重新部署的。

主要名词解释

  • Git：分布式版本控制系统。

  • GitHub：可以托管各种Git版本库，并提供一个web界面。GitHub就像是程序员们的Facebook。

  • GitHub Pages：可以被认为是用户编写的、托管在GitHub上的静态网页。由于它的空间免费稳定， 可以用于介绍托管在GitHub上的Project或者搭建网站。GitHub Pages有两种。一种是为个人或者组织的博客。一种是为项目的博客。前者一个账号只能建一个，后者，可以建很多个。

  • WordPress：是一种使用PHP语言开发的博客平台，用户可以在支持PHP和MySQL数据库的服务器上架设属于自己的网站。

静态页面工具选择

前面我们比对了GitHub Pages 和 WordPress 的优缺点，综合来看还是用GitHub Pages来搭建个人博客。

那么接下来就要考虑用什么样的静态博客框架。现在常用的有Jekyll、Hexo、Octopress等。比较Jekyll和Hexo：

  • Jekyll基于Ruby实现，使用时需要安装Ruby、Python等。Hexo基于Node.js实现。

  • Jekyll没法实现本地预览功能，需要上传到Web容器中才可预览，Hexo可以通过命令实现本地预览，并实现同步。

  • Jekyll提供的主题没有Hexo提供的色调好看。

  • 速度上jekyll生成静态站点的速度比hexo慢很多。

####配置GitHub篇

  • 首先，你要有一个GitHub的账号：

1、 用户名、邮箱、密码不能为空，且密码最少7个字符。

2、 选择一个仓库形式：第一个是无限制的免费公开仓库，第二个是每月47.44软妹币的无限制私有仓库。

3、选择一些调查问卷。第一项是描述一下自己的经验等级；第二项是选择一下你用GitHub主要是用来干嘛；第三项是你的属性；第四项是你的个人爱好。
)
4、OK，你的GitHub注册成功了。

配置Hexo篇

  • 下载安装Node

  • 安装Hexo：

    • 新建一个文件夹，比如Blog，用户存放Hexo的配置文件。在Blog文件夹安装完成Hexo后，Blog就是你的博客根目录，所有的操作都在里面进行。
    • 打开终端进入Blog夹中。
    • 执行命令 sudo npm install -g hexo
    • 输入密码
      安装成功后如下图：
    • 初始化命令 hexo init
    • 生成静态页面 hexo generate (hexo g也可以)
    • 本地启动：启动本地服务，进行文章预览调试 hexo server
    • 浏览器输入 http://localhost:4000
    • 在终端中进入Blog文件夹中， vim _config.yml

    • 在最下面改成
      deploy:

      type: git

      repo: https://github.com/FlyingPiggy/myFirstRepository.github.io.git

      branch: master

    • 执行命令 npm install hexo-deployer-git –save
